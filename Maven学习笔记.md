# Maven学习笔记

## 一、Maven的一些概念

### 一、什么是Maven

Maven是apache旗下的一款开源项目，是一款管理和构建java项目的工具（我主要先学习这个工具怎么用）。它基于项目对象模型的概念，通过一小段描述来管理项目。

apache软件基金会，是全球最大的最受欢迎的软件开源基金会，是一个非盈利组织。

### 二、Maven的作用

#### 1、依赖管理

方便快捷的管理项目依赖的资源（jar包），避免版本冲突问题。java要用到一些第三方提供的功能就需要引入响应的jar包，之前自己手动导入jar包需要先下载jar包，然后在把jar包粘到项目中，还得解决版本相匹配的问题，避免版本冲突，现在用Maven，只需要在pom.xml中配置要用到的依赖（jar包）就可以。

#### 2、统一项目结构

提供标准统一的项目结构。只要创建的是Maven项目，则在哪款开发工具上创建其项目目录结构都是一样的。可以在任意开发工具之间轻松导项目。

##### Maven项目目录结构

src->main(实际项目资源)

src->main->java(java源代码目录)

src->main->resources(配置文件目录)

src->test(测试项目资源)

src->test->java

src->test->resources

pom.xml(Maven项目核心配置文件)

target(项目编译后的字节码文件、打包后的文件都放在这里)

#### 3、项目构建

标准跨平台的自动化项目构建方式。Maven为我提供了一些快速的指令，帮助我们快速的进行项目编译、打包等操作。

### 三、Maven的组成

1、各种插件来进行项目的标准化构建

2、基于项目对象模型，通过一小段描述来管理项目。

3、依赖管理模型：在pom.xml中添加项目需要用到的依赖。

### 四、Maven的仓库

用于存储资源，管理jar包

1、本地仓库：自己计算机上的一个目录

2、中央仓库：由Maven团队维护的全球唯一的。仓库地址在国外，访问下载起来比较慢，所以很多公司都有自己的（远程仓库）私服。

3、远程仓库（私服）：一般由公司团队搭建的私有仓库，公司里有一个人下载过，其他人就不需要再到外国官网下载了，可以提高下载速度。我们自己搭建不了，用阿里云的。

### 五、Maven目录结构

1、bin存放可执行文件，Maven中用到的一些指令，比如运行、测试、打包等。

2、conf 存放配置文件，配置本地仓库，阿里云服务器，都在其中的setting文件中进行配置。

3、lib中存放的是Maven中依赖的一些jar包资源。

### 六、Maven坐标

1、什么是坐标：

Maven中的坐标是资源的唯一标识，通过该坐标可以唯一定位资源位置。

使用坐标来定义项目或引入项目中需要的依赖。

2、Maven坐标组成

groupId: 定义当前Maven项目隶属组织名称，一般为公司域名反写。

artifactId: 定义当前Maven项目名称

version:定义当前项目版本号。

## 二、依赖管理

### 一、依赖配置

```xml
<dependencies>
    <dependency>
        <groupId></groupId>
        <artifactId></artifactId>
        <version></version>
    </dependency>
</dependencies>
```



注意：如果不知道依赖的配置信息，访问https://mvnrepository.com/中搜索

### 二、依赖传递

1、Maven的依赖具有传递性

直接依赖：在pom.xml中直接通过坐标引入的依赖为直接依赖。

间接依赖：被依赖的项目如果依赖其他资源，则该项目间接依赖其它资源。

2、排除依赖

排除依赖指主动断开依赖的资源，被排除的依赖无需指定版本。

```xml
<exclusion>
    <groupId></groupId>
    <artifacteId></artifacteId>
</exclusion>
```



3、依赖范围

依赖的jar包默认情况下可以在任何地方使用。可以通过<scope></scope>设置其使用范围。

test 只测试程序内有效 例：junit

provided 主程序和测试程序有效

runtime 测试程序和打包有效。

### 三、Maven的生命周期

Maven的生命周期就是为了对所有的Maven项目的构建过程进行抽象和统一。

Maven中有三套相互独立的生命周期：每一套生命周期包含一些阶段，阶段是有顺序的，后面的阶段依赖与前面的阶段，执行后面的阶段，前面的阶段会自动执行。

clean(清理工作): 清理上次构建项目产生的字节码文件、jar包等，其中重点是clean指令。

default(核心工作：编译、测试、打包、安装、部署等)，其中重点是:

compile:编译

test：测试

package：打包

install：将打包好的jar包安装到Maven本地仓库中，以后其他项目想要依赖该项目，可以在pom.xml中写响应的坐标就可以。
